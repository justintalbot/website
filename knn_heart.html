Data Description

  1. (age) - age in years
  2. (sex) - sex (1 = male; 0 = female)
  3. (cp) - chest pain type
  • Value 1: typical angina
  • Value 2: atypical angina
  • Value 3: non-anginal pain
  • Value 4: asymptomatic
  1. (trestbps) - resting blood pressure (in mm Hg on admission to the hospital)
  2. (chol) - serum cholestoral in mg/dl
  3. (fbs) - (fasting blood sugar > 120 mg/dl) (1 = true; 0 = false)
  4. (restecg) - resting electrocardiographic results
  5. (thalach) -
  • Value 0: normal
  • Value 1: having ST-T wave abnormality (T wave inversions and/or ST elevation or depression of > 0.05 mV)
  • Value 2: showing probable or definite left ventricular hypertrophy by Estes’ criteria
  1. (exang) - exercise induced angina (1 = yes; 0 = no) 10.(oldpeak) - ST depression induced by exercise relative to rest 11.(slope) - the slope of the peak exercise ST segment
  • Value 1: upsloping
  • Value 2: flat
  • Value 3: downsloping 12.(ca) - number of major vessels (0-3) colored by flourosopy 13.(thal) - 3 = normal; 6 = fixed defect; 7 = reversable defect 14.(num) (the predicted attribute)

Clean the data

The data is already very clean, but a few minor changes are needed.

Two of the columns have “?” in them. We will remove these entries.

Also, the variable we want to predict, num, has values from 0 to 4. Where 0 means the patient does not have heart disease and 1, 2, 3, and 4 are differing levels 0f severity for heart disease. We will change this to a binary variable, having or not having, or 0 or 1.

# ca and thal have "?" entries. Remove those
heart <- heart_disease %>%
  filter(ca != "?", thal != "?")

heart$ca <- as.numeric(heart$ca)  
heart$thal <- as.numeric(heart$thal)

# The value we want to predict, num, has values 0 - 4. We only want heart disease/no heart disease
heart <- mutate(heart, num = ifelse(num >= 1, 1, 0))

Standardizing the data

KNN works by first training the dataset and grouping the data based on our variables into two groups, having and not having heart diseases. Based on this, the test set is then placed into one of the two groups based on the distance to the k (the number of nodes we choose) nearest neighbors.

We are using the Euclidian distance, or “as the crow flies”. Because of this, differences in measurement metrics can affect our results.

For example, Ted is 100cm tall and John is 1 meter. Without standardization Ted is 99 units taller than John. Therefore, we standardize all of our variables to be values between 0 and 1.

# Now we have to standardize the entries as values between 0 and 1
normalize <- function(x) {
  return ((x - min(x)) / (max(x) - min(x)))
}

heart_unstandardized <- heart
heart[,1:13] <- as.data.frame(lapply(heart[1:13], normalize))

Splitting the dataset into test and training sets

Now that our data is standardized we can split it into the test and training set.

# Seperate test and training sets
set.seed(100)

# Shuffle the dataset, call the result shuffled
n <- nrow(heart)
shuffled <- heart[sample(n),]

# Split the data in train and test
train_indices <- 1:round(0.7 * n)
train <- shuffled[train_indices, ]
test_indices <- (round(0.7 * n) + 1):n
test <- shuffled[test_indices, ]

# Pull out training and test set labels
train_labels <- train$num
test_labels <- test$num

# Copy train and test to knn_train and knn_test
knn_train <- train
knn_test <- test

# Drop num column for knn_train and knn_test
knn_train$num <- NULL
knn_test$num <- NULL

Predicting the results and creating the confusion matrix

# KNN function on dataset with a k of 5
pred <- knn(train = knn_train, test = knn_test, cl = train_labels, k = 5)

# Our confusion matrix
conf <- table(test_labels, pred)
conf

From our confusion matrix we can see the numbers of True Positives, the values that we predicted to be true and are true.

The number of True Negatives, the values that we correctly predicted to be negative.

The False Positives, the values that we predicted to be positive but are actually negative.

And finally the False Negatives. The number of values that we predicted to be negative but are actually positive.

Our overall accuracy based on K

accuracy <- sum(diag(conf))/sum(conf)
accuracy
## [1] 0.7977528
# Function to return accuracy based on supplied k
k_value <- function(x) {
  pred <- knn(train = knn_train, test = knn_test, cl = train_labels, k = x)
  conf <- table(test_labels, pred)
  accuracy <- sum(diag(conf))/sum(conf)
  accuracy
} 

k <- c(1:100)
acc <- sapply(k, k_value)
qplot(k, acc) +
  theme_bw() +
  geom_smooth(se = F, method = "loess") +
  labs(title = "Predicting Heart Disease with K-NN",
       subtitle = "Accuracy as a function of K",
       y = "accuracy")

We can see as the k, or number of matches used to match our test value to our model, increases the accuracy also increases rapidly and then levels out.

Conclusion

With a few lines of code we were able to achieve nearly 88% accuracy. The problem with an algorithm such as k-nn is that it doesn’t tell us which variables are most important in measuring heart disease. In other words, it is accurate for this dataset, but it is a bit of a black box.
